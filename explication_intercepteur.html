C'est quoi Scapy ?
Scapy est une bibliothèque Python ultra-puissante pour manipuler les paquets réseau. Un paquet c'est une petite enveloppe de données qui voyage sur le réseau — tout ce que fait ton téléphone (ouvrir une app, envoyer un message, charger une page) génère des paquets.

Scapy permet de :
→ Capturer les paquets qui circulent
→ Analyser leur contenu
→ Créer des paquets personnalisés
→ Envoyer des paquets sur le réseau

C'est l'outil numéro 1 des chercheurs en sécurité réseau.
Les imports
from scapy.all import sniff, IP, TCP, UDP, DNS, DNSQR
sniff → la fonction qui capture les paquets
IP → couche réseau (adresses IP)
TCP → protocole de transport fiable (sites web, SSH...)
UDP → protocole rapide (streaming, jeux...)
DNS → système de noms de domaine
DNSQR → requête DNS spécifique
La fonction sniff() — écouter le réseau
sniff(
  filter=f"host {IP_TELEPHONE}", # filtre par IP
  prn=analyser_paquet, # fonction appelée à chaque paquet
  store=0 # ne stocke pas en mémoire
)
sniff() est une boucle infinie qui écoute tout le trafic réseau. Pour chaque paquet capturé elle appelle automatiquement la fonction analyser_paquet.

filter="host X" utilise la syntaxe BPF (Berkeley Packet Filter) pour ne capturer que les paquets de/vers une IP spécifique — sinon on verrait tout le trafic du réseau.

store=0 → ne garde pas les paquets en mémoire RAM, sinon après des milliers de paquets le PC manquerait de mémoire.
Vérifier si un paquet contient une couche
if paquet.haslayer(IP):
  src = paquet[IP].src # IP source
  dst = paquet[IP].dst # IP destination
Un paquet réseau est comme une poupée gigogne — il contient plusieurs couches empilées. haslayer(IP) vérifie si la couche IP est présente. paquet[IP].src lit l'adresse source dans cette couche.
Les protocoles réseau
Chaque type de communication utilise un protocole différent. C'est comme différents langages selon ce qu'on veut dire.
DNS
→
Traduit un nom de site en IP. Ex : "google.com" → "142.250.74.46"
TCP
→
Transport fiable — sites web, emails, SSH. Garantit que les données arrivent
UDP
→
Transport rapide — streaming, jeux, appels vidéo. Priorité à la vitesse
HTTPS 443
→
Web sécurisé — contenu chiffré, on voit la destination mais pas le contenu
Identifier le service par le port
services = {
  80: "HTTP",
  443: "HTTPS",
  22: "SSH",
  25: "Email",
}
service = services.get(port, f"Port {port}")
.get(clé, valeur_par_défaut) cherche une clé dans un dictionnaire. Si elle n'existe pas il retourne la valeur par défaut — ici le numéro de port brut.
Le DNS — voir quels sites visite ton téléphone
if paquet.haslayer(DNS) and paquet.haslayer(DNSQR):
  domaine = paquet[DNSQR].qname.decode("utf-8").rstrip(".")
  print(f"🌐 DNS → {domaine}")
C'est la partie la plus intéressante ! Avant d'aller sur un site, ton téléphone envoie une requête DNS pour trouver son IP. En capturant ces requêtes on peut voir tous les domaines que visite ton téléphone — même ceux des apps en arrière-plan !

.decode("utf-8") → convertit les bytes en texte lisible
.rstrip(".") → supprime le point final que DNS ajoute automatiquement
Lien avec la vraie cybersécurité
Ce que tu fais s'appelle du packet sniffing ou network sniffing. C'est une technique fondamentale utilisée par :

→ Les administrateurs réseau pour diagnostiquer des problèmes
→ Les pentesters pour analyser le trafic d'une cible
→ Les hackers malveillants pour voler des données sur des réseaux non sécurisés

C'est pour ça que le HTTPS est crucial — même si quelqu'un intercepte tes paquets, le contenu est chiffré et illisible. Tu verras le domaine mais pas ce que tu envoies.

Sur un réseau Wi-Fi public non sécurisé, n'importe qui peut faire ce que tu fais maintenant — c'est pourquoi il ne faut jamais se connecter à son compte bancaire sur un Wi-Fi public !
