C'est quoi ReportLab ?
ReportLab est une bibliothèque Python qui permet de créer des fichiers PDF depuis du code. Au lieu d'ouvrir Word et de taper du texte, tu décris le document en Python et il le génère automatiquement.

C'est utilisé en entreprise pour générer des factures, des rapports, des bulletins de paie automatiquement.
Les imports — chaque outil a son rôle
from reportlab.lib.pagesizes import A4 # taille de page
from reportlab.lib import colors # couleurs
from reportlab.lib.styles import ... # styles de texte
from reportlab.platypus import Paragraph # paragraphe
from reportlab.platypus import Table # tableau
from reportlab.platypus import Spacer # espace vide
from reportlab.platypus import HRFlowable # ligne horizontale
ReportLab est divisé en plusieurs modules. platypus (oui c'est le vrai nom !) gère la mise en page automatique — il place les éléments les uns après les autres comme un traitement de texte.
Créer le document
doc = SimpleDocTemplate(nom_fichier, pagesize=A4,
    rightMargin=2*cm, leftMargin=2*cm)
doc.build(contenu)
SimpleDocTemplate crée le fichier PDF avec les marges définies. doc.build(contenu) à la fin assemble tout et génère le fichier — c'est la dernière instruction qui déclenche la création.
Les styles — définir l'apparence du texte
titre_style = ParagraphStyle(
  "titre",
  fontSize=22,
  textColor=colors.HexColor("#00ff88"),
  alignment=TA_CENTER,
  fontName="Helvetica-Bold"
)
Un style c'est un ensemble de règles de mise en forme qu'on applique à un texte. On définit les styles une fois en haut, et on les réutilise partout dans le document.

fontSize → taille de la police
textColor → couleur du texte
alignment → alignement (centre, gauche, droite)
fontName → police utilisée

La liste contenu — empiler les éléments
contenu = []
contenu.append(Paragraph("🔐 CyberMap 3D", titre_style))
contenu.append(Spacer(1, 0.5*cm))
contenu.append(HRFlowable(...))
contenu.append(Table(...))
contenu est une simple liste Python. On y ajoute des éléments dans l'ordre d'apparition dans le PDF :

Paragraph → un bloc de texte
Spacer → un espace vide entre les éléments
HRFlowable → une ligne horizontale de séparation
Table → un tableau avec des lignes et colonnes

À la fin, doc.build(contenu) prend cette liste et construit le PDF dans l'ordre.
Le tableau — afficher les données en grille
resume_data = [
  ["Indicateur", "Valeur"], # en-tête
  ["Machines détectées", "7"],
  ["Ports ouverts", "4"],
  ["Vulnérabilités", "0"],
]
table = Table(resume_data, colWidths=[10*cm, 6*cm])
Un tableau dans ReportLab c'est une liste de listes — chaque liste intérieure est une ligne, chaque élément est une cellule. colWidths définit la largeur de chaque colonne en centimètres.
Le style du tableau
table.setStyle(TableStyle([
  ("BACKGROUND", (0,0), (-1,0), colors.HexColor("#0a0a1a")),
  ("TEXTCOLOR", (0,0), (-1,0), colors.HexColor("#00ff88")),
  ("GRID", (0,0), (-1,-1), 0.5, colors.HexColor("#cccccc")),
]))
Les coordonnées (0,0) signifient colonne 0, ligne 0. (-1,0) signifie "dernière colonne, ligne 0". Donc (0,0),(-1,0) = toute la première ligne.

(-1,-1) = dernière colonne, dernière ligne → donc (0,0),(-1,-1) = tout le tableau.
Le calcul du résumé
total_machines = len(machines)
total_ports = sum(len(m.get("ports", [])) for m in machines)
total_vulns = sum(len(m.get("vulnerabilites", [])) for m in machines)
len(machines) → compte le nombre de machines.

La ligne sum(len(...) for m in machines) c'est une compréhension de générateur — une boucle condensée sur une ligne. Elle calcule le nombre de ports de chaque machine et les additionne tous. C'est l'équivalent de :

total = 0
for m in machines:
  total += len(m["ports"])

.get("ports", []) → si la clé "ports" n'existe pas, retourne une liste vide plutôt que de planter.
