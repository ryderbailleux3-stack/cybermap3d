Explication du code HTML et Three.js

Concept HTML — la structure de la page
HTML c'est le squelette de toute page web. Il dit au navigateur ce qui existe sur la page — un titre, une image, un bouton. Chaque élément est une balise entre < >.
<!DOCTYPE html> — dit que c'est une page web
<html> — tout le contenu va ici
<head> — infos invisibles (titre, style)
  <title>CyberMap 3D</title>
</head>
<body> — ce que l'utilisateur voit
  <div id="info">CyberMap 3D</div>
</body>
</html>
Chaque balise ouvrante <div> doit avoir une balise fermante </div>. C'est comme des parenthèses qui s'ouvrent et se ferment.
Balise script Importer Three.js
Three.js est une bibliothèque JavaScript qui gère toute la 3D. On l'importe depuis internet avec une balise <script src="..."> — comme un import en Python.
# Python
import json
// JavaScript (équivalent)
<script src="three.min.js"></script>
En Python on dit import. En HTML on dit <script src>. C'est la même idée : charger du code externe pour s'en servir.
Concept CSS — le style visuel
Le CSS dit au navigateur comment ça doit avoir l'air — couleurs, tailles, positions. Sans CSS, tout serait blanc avec du texte noir.
* {
  margin: 0; /* supprime les espaces par défaut */
  padding: 0; /* supprime les marges internes */
}
body {
  background: #0a0a1a; /* fond noir/bleu foncé */
}
#info {
  position: absolute; /* flotte au-dessus */
  color: #00ff88; /* vert néon */
  font-family: monospace;
}
#info cible l'élément avec id="info". Le # veut dire "l'élément avec cet id". C'est comme chercher une variable par son nom.
Concept JavaScript — le comportement
HTML = structure, CSS = apparence, JavaScript = action. C'est le seul langage que le navigateur comprend nativement. Il ressemble beaucoup à Python.
// Python vs JavaScript
# Python
age = 17
if age > 18:
    print("majeur")
// JavaScript
const age = 17
if (age > 18) {
  console.log("majeur")
}
Les différences principales : JS utilise const/let au lieu de rien, les blocs sont entre { } au lieu d'indentation, et console.log remplace print.
Boucle forEach — l'équivalent du for Python
# Python
for machine in machines:
    print(machine["nom"])
// JavaScript
machines.forEach(machine => {
  console.log(machine.nom)
})
C'est la même logique. La flèche => en JS s'appelle une "fonction fléchée" — c'est l'équivalent du : après le for en Python.
Concept Les 3 éléments indispensables de Three.js
Pour afficher quelque chose en 3D, Three.js a besoin de 3 choses — exactement comme un vrai tournage de film.
Scene
→
Le plateau de tournage. Tout ce qui existe (cubes, lumières) est ajouté ici avec scene.add()
Camera
→
L'œil qui regarde la scène. On la place avec camera.position.z = 10
Renderer
→
Le projecteur qui dessine la scène sur l'écran à chaque frame
Cubes Comment un cube est créé
// 1 — La forme (géométrie)
const geo = new THREE.BoxGeometry(1, 1, 1)
// 2 — La couleur (matériau)
const mat = new THREE.MeshPhongMaterial({ color: 0xff2222 })
// 3 — L'objet (forme + couleur)
const cube = new THREE.Mesh(geo, mat)
// 4 — Placer dans la scène
cube.position.set(-4, 0, 0)
scene.add(cube)
0xff2222 c'est du rouge en hexadécimal. En CSS tu écrirais #ff2222 — c'est la même chose, juste avec 0x devant pour JavaScript.
Concept La boucle d'animation
Pour que la scène tourne, on crée une boucle infinie qui se répète 60 fois par seconde. À chaque répétition on tourne un peu la scène et on la redessine.
function animer() {
  requestAnimationFrame(animer) // rappelle animer() au prochain frame
  scene.rotation.y += 0.005 // tourne de 0.005 radian
  renderer.render(scene, camera) // redessine
}
animer() // lance la boucle
requestAnimationFrame c'est l'équivalent de time.sleep en Python mais pour les animations — il synchronise avec le rafraîchissement de l'écran (60fps).
Architecture Comment Python et Three.js communiquent
Les deux langages ne se parlent pas directement. Le fichier JSON est le messager entre eux.
Python
→
Simule l'attaque, génère les données du réseau
reseau.json
→
Stocke l'état du réseau (qui est infecté, les IPs...)
Three.js
→
Lit le JSON et place les machines en 3D avec les bonnes couleurs
C'est exactement l'architecture qu'utilisent les vrais outils de cybersécurité — un backend qui analyse, un frontend qui visualise. Tu construis quelque chose de professionnel.
