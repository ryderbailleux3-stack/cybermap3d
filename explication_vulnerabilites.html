C'est quoi une CVE ?
CVE = Common Vulnerabilities and Exposures. C'est une base de données mondiale qui recense toutes les failles de sécurité connues dans les logiciels.

Chaque faille a un identifiant unique : CVE-2017-0144
→ L'année = 2017
→ Le numéro = 0144

CVE-2017-0144 c'est la faille WannaCry — elle exploitait le port 445 (SMB) que ton PC a ouvert ! En 2017 elle a infecté 230 000 ordinateurs dans 150 pays.

La base CVE est maintenue par le gouvernement américain (NIST) et est publique — tout le monde peut la consulter, les défenseurs comme les attaquants.
L'API NIST — interroger la base CVE
url = f"https://services.nvd.nist.gov/rest/json/cves/2.0?keywordSearch={service}+{version}"
reponse = requests.get(url, timeout=10)
data = reponse.json()
Une API c'est une interface qui permet à deux programmes de communiquer. Ici on envoie une requête HTTP au serveur du NIST (comme quand ton navigateur charge une page web), et il nous répond avec un JSON contenant les CVE trouvées.

requests.get(url) → envoie la requête
reponse.json() → convertit la réponse en dictionnaire Python
timeout=10 → abandonne si le serveur ne répond pas en 10 secondes
Ce que renvoie l'API
{ "vulnerabilities": [
  { "cve": {
    "id": "CVE-2021-41773",
    "descriptions": [{ "value": "Path traversal..." }],
    "metrics": { "cvssMetricV31": [...] }
  }}
]}
C'est un JSON imbriqué — des dictionnaires dans des listes dans des dictionnaires. On navigue dedans avec des crochets comme data["vulnerabilities"][0]["cve"]["id"].
Le score CVSS — mesurer la gravité d'une faille
Chaque CVE a un score de 0 à 10 qui mesure sa dangerosité :

🟢 0.0 – 3.9 → LOW (faible)
🟡 4.0 – 6.9 → MEDIUM (moyen)
🟠 7.0 – 8.9 → HIGH (élevé)
🔴 9.0 – 10.0 → CRITICAL (critique)

WannaCry avait un score de 9.3 / CRITICAL.
Comment on récupère le score
if "cvssMetricV31" in cve["metrics"]:
  score = cve["metrics"]["cvssMetricV31"][0]["cvssData"]["baseScore"]
  gravite = cve["metrics"]["cvssMetricV31"][0]["cvssData"]["baseSeverity"]
elif "cvssMetricV2" in cve["metrics"]:
  score = cve["metrics"]["cvssMetricV2"][0]["cvssData"]["baseScore"]
Il existe deux versions du score CVSS — V3.1 (plus récent) et V2 (plus ancien). On vérifie d'abord V3.1, et si absent on prend V2. Le [0] prend le premier résultat de la liste.
La boucle principale — logique du programme
for machine in machines: # pour chaque machine
  for port in machine["ports"]: # pour chaque port ouvert
    service = port["service"]
    version = port["version"]
    cves = chercher_cve(service, version) # interroge NIST
    machine["vulnerabilites"] = cves # stocke les résultats
C'est une double boucle — une boucle dans une boucle. Pour chaque machine on parcourt ses ports, et pour chaque port on interroge la base CVE. Si une machine a 3 ports, on fait 3 requêtes à l'API.
Try / Except — gérer les erreurs
try:
  reponse = requests.get(url, timeout=10)
  data = reponse.json()
except Exception as e:
  return []
Le try/except c'est comme un filet de sécurité. Si quelque chose se passe mal dans le bloc try (pas de connexion internet, serveur lent, réponse invalide), au lieu de planter tout le programme Python passe dans le bloc except et retourne une liste vide.

Sans ça, une seule erreur réseau ferait crasher tout le scan.
