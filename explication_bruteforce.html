C'est quoi le bruteforce SSH ?
Le bruteforce (force brute) c'est essayer tous les mots de passe possibles jusqu'à trouver le bon. C'est l'attaque la plus simple et la plus courante contre SSH.

SSH (Secure Shell) permet de se connecter à distance à un serveur Linux. Si un serveur SSH est exposé sur internet, des bots automatiques essaient des milliers de mots de passe par seconde en permanence.

Notre simulation teste :
→ 4 utilisateurs × 20 mots de passe = 80 tentatives
→ Un vrai outil comme Hydra teste des millions de combinaisons
Bruteforce vs Dictionnaire
Bruteforce pur
→
Teste toutes les combinaisons possibles (aaa, aab, aac...) — très lent
Dictionnaire
→
Teste une liste de mots courants — beaucoup plus efficace en pratique
Hybride
→
Dictionnaire + variations (hugo123, Hugo!, HUGO...) — le plus efficace
La double boucle — tester toutes les combinaisons
for utilisateur in UTILISATEURS: # root, admin, user, hugo
  for mot_de_passe in DICTIONNAIRE: # 20 mots de passe
    succes = simuler_tentative(utilisateur, mot_de_passe)
    if succes:
      print("ACCES OBTENU !")
C'est une double boucle — on a déjà vu ça dans le scanner de ports ! Pour chaque utilisateur on essaie tous les mots de passe. Si on a 4 utilisateurs et 20 mots de passe → 4 × 20 = 80 tentatives.

Un vrai attaquant avec une liste de 10 000 utilisateurs et 1 million de mots de passe ferait 10 milliards de tentatives.
Pourquoi on simule sans vraie connexion
def simuler_tentative(utilisateur, mot_de_passe):
  time.sleep(0.2) # simule le délai réseau
  if utilisateur == "admin" and mot_de_passe == "admin":
    return True # succes simulé
  return False
On simule les tentatives au lieu de faire de vraies connexions SSH pour deux raisons :

1. Légalité — faire de vraies tentatives de connexion sur un serveur sans autorisation est illégal, même sur localhost

2. Pédagogie — comprendre la logique suffit. Le vrai code avec Paramiko (bibliothèque SSH Python) serait identique, juste avec une vraie connexion à la place de sleep()
Comment se protéger du bruteforce
fail2ban
→
Bloque automatiquement une IP après X tentatives échouées
Clé SSH
→
Remplace le mot de passe par une clé cryptographique — bruteforce impossible
Port custom
→
Changer SSH du port 22 au port 2222 réduit 90% des scans automatiques
2FA
→
Double authentification — même avec le bon mot de passe, besoin d'un code
Lien avec la vraie cybersécurité
Chaque serveur SSH exposé sur internet reçoit des milliers de tentatives de bruteforce par jour — c'est automatique, des bots scannent en permanence toute l'IPv4 mondiale.

L'outil Hydra peut tester jusqu'à 50 000 mots de passe par seconde en parallèle. Avec une liste de 10 millions de mots de passe courants, un serveur avec le mot de passe "admin" se fait compromettre en quelques secondes.

C'est pour ça que les admins système utilisent toujours :
→ Des clés SSH au lieu de mots de passe
→ fail2ban pour bloquer les IPs suspectes
→ Un port SSH non standard

En comprenant comment fonctionne le bruteforce, tu comprends pourquoi ces protections sont indispensables.
