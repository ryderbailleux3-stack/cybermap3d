Variables globales — partagées entre les fonctions
let attaqueInterval = null
let attaqueEnCours = false
Ces deux variables sont déclarées en dehors des fonctions — elles sont donc accessibles par toutes les fonctions du programme. C'est ce qu'on appelle des variables globales.

attaqueInterval → stocke la référence du setInterval pour pouvoir l'arrêter plus tard avec clearInterval. Sans ça, impossible de stopper l'attaque !

attaqueEnCours → un booléen qui dit si l'attaque est active ou non. true = en cours, false = stoppée.
Pourquoi globales ?
Si ces variables étaient à l'intérieur de lancerAttaque(), la fonction activerParefeu() ne pourrait pas y accéder — chaque fonction a son propre espace mémoire.

En les mettant dehors, les deux fonctions partagent les mêmes variables — comme un tableau d'affichage visible par tout le monde.
Ce qui a changé dans lancerAttaque()
// Nouveau — affiche le bouton pare-feu
getElementById("btn-parefeu").style.display = "block"
// Nouveau — marque l'attaque comme active
attaqueEnCours = true
// Nouveau — stocke le setInterval
attaqueInterval = setInterval(() => { ... }, 1000)
// Nouveau — quand terminée
attaqueEnCours = false
La différence principale c'est qu'on sauvegarde le setInterval dans attaqueInterval. Avant on le lançait sans le stocker — impossible à arrêter. Maintenant on a une "poignée" pour le stopper quand on veut avec clearInterval(attaqueInterval).
activerParefeu() — stopper l'attaque
clearInterval(attaqueInterval) // stoppe la propagation
attaqueEnCours = false // marque comme stoppée
meshes.forEach(m => {

    const hex = child.material.color.getHex()
    if (hex === 0xff2222) { // rouge = infecté

      // passe en vert
    }

})
clearInterval stoppe immédiatement la boucle d'infection — plus aucune machine ne sera infectée.

Ensuite on parcourt toutes les machines avec forEach et traverse, et on regarde la couleur actuelle avec getHex() :

→ Rouge (0xff2222) = déjà infectée → on la passe en orange (compromise mais stoppée)
→ Autre couleur = saine → on la remet en vert
Le système de couleurs — lire et changer
// Lire la couleur actuelle
const hex = child.material.color.getHex()
// Comparer
if (hex === 0xff2222) { ... }
// Changer la couleur
child.material = new THREE.MeshPhongMaterial({ color: 0xff8800 })
getHex() retourne la couleur actuelle du matériau en hexadécimal — comme lire l'étiquette d'une boîte.

Pour changer la couleur on crée un nouveau matériau — c'est la façon de faire dans Three.js.

Le lien avec la vraie cybersécurité
Ce que tu simules c'est exactement ce qui se passe lors d'une vraie attaque réseau :

Sans pare-feu → le virus se propage librement de machine en machine jusqu'à tout infecter.

Avec pare-feu → dès qu'une activité suspecte est détectée, le pare-feu isole les machines infectées et bloque la propagation. Les machines déjà infectées restent compromises (orange) mais les autres sont sauvées (vert).

C'est le principe du containment en cybersécurité — limiter les dégâts en isolant rapidement la menace. Plus vite tu actives le pare-feu, moins de machines sont perdues !
