C'est quoi un hash ?
Un hash c'est une empreinte numérique d'un texte. Quand tu entres un mot de passe sur un site, le site ne stocke jamais le mot de passe en clair — il stocke son hash.

C'est une fonction à sens unique :
→ "admin" → 21232f297a57a5a743894a0e4a801fc3
→ Impossible de retrouver "admin" depuis le hash

Propriétés d'un bon hash :
→ Le même texte donne toujours le même hash
→ Un seul caractère différent change tout le hash
→ Impossible de revenir en arrière (théoriquement)
Pourquoi c'est utile en sécurité ?
Si un hacker vole la base de données d'un site, il ne voit que des hashs — pas les vrais mots de passe. Pour se connecter il doit retrouver le mot de passe original, ce qui est difficile... mais pas impossible pour les mots de passe faibles !
Les différents algorithmes de hash
MD5
→
32 caractères. Rapide mais obsolète et non sécurisé — à éviter !
SHA1
→
40 caractères. Également obsolète depuis 2017
SHA256
→
64 caractères. Standard actuel — beaucoup plus sécurisé
bcrypt
→
Conçu spécialement pour les mots de passe — le meilleur choix
Comment créer un hash en Python
import hashlib
h = hashlib.new("md5")
h.update("admin".encode("utf-8"))
resultat = h.hexdigest()
# → "21232f297a57a5a743894a0e4a801fc3"
encode("utf-8") convertit le texte en bytes — hashlib travaille avec des bytes pas du texte. hexdigest() retourne le hash en hexadécimal lisible.
L'attaque par dictionnaire
Une attaque par dictionnaire c'est simple — on prend une liste de mots de passe courants, on calcule leur hash, et on compare avec le hash cible.

Si le hash correspond → on a trouvé le mot de passe !

C'est efficace car les gens utilisent des mots de passe prévisibles. Les 10 mots de passe les plus utilisés au monde sont toujours les mêmes : 123456, password, admin...
La logique de comparaison
for mot in dictionnaire:
  hash_essai = hasher(mot)
  if hash_essai == hash_cible:
    print(f"Trouvé : {mot}")

On calcule le hash de chaque mot du dictionnaire et on compare avec le hash cible. Si les deux hashs sont identiques → le mot de passe est trouvé ! C'est aussi simple que ça.

for i, mot in enumerate(dictionnaire):

enumerate() donne un numéro à chaque élément — tu as déjà vu ça dans scanner_ports.py ! Ici on l'utilise pour afficher le numéro de tentative.
time.time() — mesurer la durée
debut = time.time()
# ... craquage ...
duree = time.time() - debut
print(f"Trouvé en {duree:.3f} secondes")
time.time() retourne le nombre de secondes depuis le 1er janvier 1970. En soustrayant le temps de début du temps de fin on obtient la durée exacte. :.3f affiche 3 décimales.
Lien avec la vraie cybersécurité
En 2012, LinkedIn s'est fait voler 117 millions de mots de passe hashés en SHA1. En quelques jours, des hackers avaient cracké la majorité car les utilisateurs avaient des mots de passe faibles.

Des outils comme Hashcat ou John the Ripper peuvent tester des milliards de mots de passe par seconde sur GPU — ton dictionnaire de 35 mots se craque en microsecondes !

Comment se protéger :
→ Utiliser un mot de passe long et aléatoire (+16 caractères)
→ Ne jamais réutiliser le même mot de passe
→ Utiliser un gestionnaire de mots de passe (Bitwarden, KeePass)
→ Activer l'authentification à deux facteurs partout

Les sites modernes utilisent bcrypt ou Argon2 qui sont intentionnellement lents — même avec un GPU puissant ça prend des années à craquer.
